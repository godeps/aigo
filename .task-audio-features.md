# Task: Add Audio Capabilities to aigo

## Context

aigo is an agent-native Go SDK for multimodal media generation. It currently supports:
- Image generation (Qwen, Wan, Z-Image via aliyun engine)
- Video generation (Wan via aliyun engine)
- TTS (Qwen3 TTS via aliyun engine)
- Voice design (Qwen via aliyun engine)

This project has **zero external dependencies** — only Go stdlib. Follow this pattern strictly.

## What to Implement

### 1. Speech-to-Text (STT) — Qwen/Paraformer on Bailian

Add speech recognition to the existing `engine/aliyun` package.

**API Details:**
- Submit transcription: `POST {baseURL}/services/audio/asr/transcription`
- Async polling: `POST {baseURL}/tasks/{task_id}` (reuse existing poll infrastructure)
- Model: `paraformer-v2` (recommended), `fun-asr`, `qwen3-asr-flash`
- Requires header: `X-DashScope-Async: enable`

**Submit Request Body:**
```json
{
  "model": "paraformer-v2",
  "input": {
    "file_urls": ["https://example.com/audio.wav"]
  },
  "parameters": {
    "channel_id": [0],
    "language_hints": ["zh", "en"]
  }
}
```

**Poll Response (when SUCCEEDED):**
```json
{
  "output": {
    "task_status": "SUCCEEDED",
    "results": [{
      "file_url": "...",
      "transcription_url": "https://...result.json"
    }]
  }
}
```

**Transcription result (from transcription_url):**
```json
{
  "transcripts": [{
    "channel_id": 0,
    "text": "识别的文本内容",
    "sentences": [...]
  }]
}
```

**What to create:**
- `engine/aliyun/internal/asr/transcription.go` — STT implementation
- Add models `ModelParaformerV2`, `ModelFunASR`, `ModelQwenASRFlash` to `engine/aliyun/engine.go`
- Add to `modelTable` mapping
- Update `Capabilities()` to include STT models with media type "audio"
- `STTOptions` in `client.go` (if not exists — check first)
- `engine/aliyun/internal/graphx/extract.go` — add STT field extraction helpers (audio URL, language hints, channel ID, etc.)
- `ierr` — add `ErrMissingAudioURL` sentinel if not exists
- Example: `examples/aliyun_qwen_stt/main.go`

### 2. Music Generation — MiniMax

Create a **new engine** `engine/minimax` for MiniMax music generation.

**API Details:**
- Endpoint: `POST https://api.minimaxi.com/v1/music_generation`
- Auth: `Authorization: Bearer {api_key}`
- Models: `music-2.6` (recommended), `music-cover`, `music-2.6-free`, `music-cover-free`

**Request Body:**
```json
{
  "model": "music-2.6",
  "prompt": "独立民谣,忧郁,内省,独自漫步,咖啡馆",
  "lyrics": "[verse]\n...\n[chorus]\n...",
  "stream": false,
  "output_format": "url",
  "is_instrumental": false,
  "lyrics_optimizer": false,
  "audio_setting": {
    "sample_rate": 44100,
    "bitrate": 256000,
    "format": "mp3"
  }
}
```

**Response:**
```json
{
  "data": {
    "status": 2,
    "audio": "hex_encoded_audio_or_url"
  },
  "base_resp": {
    "status_code": 0,
    "status_msg": "success"
  },
  "extra_info": {
    "music_duration": 25364,
    "music_sample_rate": 44100
  }
}
```

Note: music-generation is a synchronous API (no polling needed).

**What to create:**
- `engine/minimax/engine.go` — MiniMax engine (follow patterns from engine/openai or engine/newapi)
- `engine/minimax/engine_test.go` — basic tests
- `engine/minimax/internal/graph/extract.go` — field extraction from workflow.Graph
- `engine/minimax/doc.go` — package documentation
- Add `MusicOptions` struct to `client.go`
- Add `GenerateMusic()` tool definition to `tooldef/tooldef.go`
- Add `TranscribeAudio()` tool definition to `tooldef/tooldef.go` (if not already present)
- Example: `examples/minimax_music/main.go`

## Design Guidelines

1. **Follow existing patterns exactly.** Study `engine/aliyun/internal/audiogen/tts.go` and `engine/aliyun/engine.go` for the aliyun STT implementation. Study `engine/openai/engine.go` and `engine/newapi/qwen.go` for the minimax music engine.

2. **Zero external dependencies.** Only Go stdlib.

3. **Use workflow.Graph** for input extraction. Create appropriate `*Input` node types (like `AudioOptions`, `VoiceDesignInput`, `MusicInput`).

4. **Error handling.** Use `engine/aigoerr` for classified errors. Create sentinel errors in a new `ierr` package or reuse existing.

5. **Polling for STT.** The aliyun engine already has `runtime.RT` with `WaitForCompletion` and `PollInterval`. The transcription API uses a task-based async pattern similar to video generation. Study `engine/aliyun/internal/vidgen/text_to_video.go` and `engine/aliyun/internal/runtime/runtime.go` for the polling pattern.

6. **Update README.md** and `README.zh-CN.md` with the new capabilities.

7. **Tests.** Add basic unit tests following existing test patterns.
